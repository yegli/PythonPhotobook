## File Organization Logic Flow

```
1. Count files to process (single directory walk)
2. For each file:
   - Determine if it's an image (by extension)
   - If image:
     * Get file date (from metadata or filename)
     * Build path: dest/Images/YYYYMM_Month/YYYY-MM-DD/filename
     * Convert HEIC if needed
   - If not image:
     * Route to: dest/Unsorted_Files/EXTENSION/filename
   - Copy file (or skip if exists)
```

## Common Tasks

### Adding New Feature
1. Add config to `modules/config.py` if needed
2. Create focused function with type hints
3. Use `logger` not `print`
4. Handle specific exceptions
5. Update `ARCHITECTURE.md`

### Fixing Bug
1. Check logs (enable DEBUG level)
2. Test with `--dry-run` first
3. Add error handling if missing
4. Don't break backward compatibility

### Refactoring
- Keep `image_sorting()` wrapper for compatibility
- Break large functions into small ones
- Add type hints
- Remove comments that state obvious
- Use pathlib over os.path

## Important Notes

⚠️ **Backward Compatibility**: `image_sorting()` function maintained as wrapper
⚠️ **No Tests Yet**: Full unit test suite recommended for future
⚠️ **Single Walk**: Don't add duplicate directory traversals
⚠️ **Config First**: Add new constants to config.py, not inline

## Anti-Pattern Detection

If you see this in code:

- `print()` → Should be `logger.info()`
- `except Exception:` → Should be specific exception
- Long function (>50 lines) → Should be split
- Commented code → Should be removed
- Hardcoded string → Should be in config
- `os.path.join()` → Should use pathlib `/`
- Custom logger class → Should use `logging` module
